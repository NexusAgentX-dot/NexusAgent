# 13 Hero Use Case

## Purpose
This file freezes the single hero use case for the hackathon MVP.
All demo copy, UI examples, API demo payloads, and README examples should align to this use case unless explicitly marked otherwise.

## Builder Story
A builder brings a market-aware agent into NexusAgent through the onboarding surface using an OKX-friendly integration path.

## Showcase Intent
`Check the OKB market signal and, if the run is approved, execute a bounded stablecoin proof transfer on X Layer.`

## Why This Use Case
This use case is strong because it is:
- understandable to judges
- narrow enough for a hackathon MVP
- compatible with a real X Layer proof artifact
- compatible with a payment-relevant event
- aligned with the product thesis of builder onboarding plus agent workflow execution

## What It Proves
1. An external agent can be brought into the system
2. A specialized agent can fetch context
3. Another agent can decide whether to proceed
4. A bounded execution can occur
5. A payment-related event can be shown
6. A real X Layer artifact can be recorded

## Constraints
- this is a controlled workflow, not unconstrained autonomous finance
- the stablecoin proof transfer exists to demonstrate settlement and proof, not to claim a full trading product
- any x402-related payment event must be labeled accurately as live or compatible/demo
